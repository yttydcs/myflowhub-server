# Server：升级 myflowhub-core 到 v0.3.0（对齐 Pipe 抽象重大变更）

## 背景 / 目标
- Core `v0.3.0` 引入 Pipe 抽象并新增 `listener/multi_listener`，属于重大变更（Breaking Change）。
- Server `main` 已合入对 `multi_listener` 的引用，但 `go.mod` 仍固定 `myflowhub-core v0.2.1`，导致在 `GOWORK=off` 下无法编译/测试。

目标：
- 将 Server 的 `myflowhub-core` 依赖升级到 `v0.3.0`，确保默认构建模式可用。

## 具体变更
- `go.mod`：`github.com/yttydcs/myflowhub-core v0.2.1` → `v0.3.0`
- `go.sum`：更新校验和

## 测试与验证
- `GOWORK=off go test ./... -count=1 -p 1` ✅

## 潜在影响
- 若上游/下游仍固定 Core `v0.2.1`，将无法编译当前 Server `main`（已开始依赖 `multi_listener`）。

## 回滚方案
- revert 本次依赖升级提交（同时需回滚此前引入 `multi_listener` 的代码改动，否则仍无法编译）。
